# Сабирање

Ово је први асемблерски пример у уводном делу курса. На њему се најједноставније виде:

- како аргументи функције стижу у регистрима `edi` и `esi`
- како се повратна вредност враћа преко регистра `eax`
- да за једноставну leaf функцију стек често уопште није потребан

## Шта програм ради

Програм из `main.cpp` учитава два цела броја, позива функцију `saberi` и исписује њихов збир.

Референтна C++ верзија функције изгледала би овако:

```cpp
int saberi(int a, int b) {
    return a + b;
}
```

## Датотеке

- `main.cpp` садржи C++ код који позива функцију `saberi`
- `saberi.s` је минимална верзија функције, без стек оквира
- `saberi-sa-stek-okvirom.s` је верзија која прави стек оквир, смешта аргументе у меморију и затим их чита назад

## Веза са C++ кодом

У `main.cpp` стоји декларација

```cpp
extern "C" int saberi(int a, int b);
```

`extern "C"` је важан зато што говори C++ компајлеру да име функције не сме да буде измењено C++ name mangling-ом. Захваљујући томе симбол у асемблеру заиста остаје `saberi`, па линкер може да повеже C++ позив са асемблерском имплементацијом.

## Шта треба посматрати у асемблеру

- Први аргумент функције `a` стиже у регистар `edi`.
- Други аргумент функције `b` стиже у регистар `esi`.
- Пошто `int` заузима 4 бајта, овде природно користимо 32-битне регистре `edi`, `esi` и `eax`.
- Инструкција `add edi, esi` сабира вредности.
- Пошто се повратна вредност враћа преко регистра `eax`, резултат на крају премештамо у `eax`.

Ово је добар пример зато што нема додатних компликација: нема локалних променљивих, нема приступа меморији и нема позива других функција.

## Зашто постоје две верзије

Верзија `saberi.s` је педагошки најчистија јер показује да једноставна функција понекад може да стане у свега неколико инструкција:

```asm
saberi:
    add edi, esi
    mov eax, edi
    ret
```

Ово је пример **leaf функције**: она не позива друге функције и не користи локалне променљиве на стеку. Зато јој нису потребни ни `push rbp`, ни `mov rbp, rsp`, ни резервисање простора на стеку.

Верзија `saberi-sa-stek-okvirom.s` уводи стандардни улазак у функцију:

```asm
push rbp
mov rbp, rsp
```

После тога аргументи се уписују у стек оквир:

```asm
sub rsp, 16
mov [rbp-4], edi
mov [rbp-8], esi
```

и затим се поново читају из меморије:

```asm
mov edx, [rbp-4]
mov eax, [rbp-8]
add eax, edx
```

Овде су офсети `-4` и `-8` природни зато што `int` заузима по 4 бајта. Иако за два `int` аргумента треба свега 8 бајтова простора, у примеру резервишемо 16 бајтова да бисмо задржали исправно поравнање стека.

Та верзија није потребна за само сабирање, али је корисна као припрема за касније примере у којима функција:

- има локалне променљиве
- чува регистре
- позива друге функције
- мора да води рачуна о стеку и његовом поравнању

## Превођење

Минимална верзија:

```sh
g++ main.cpp saberi.s
```

Верзија са стек оквиром:

```sh
g++ main.cpp saberi-sa-stek-okvirom.s
```

## Покретање

После превођења програм се покреће на уобичајен начин:

```sh
./a.out
```

Пример интеракције:

```text
3 5
8
```
