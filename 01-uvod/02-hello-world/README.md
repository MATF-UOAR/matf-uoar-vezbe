# Hello world

Овај пример долази после сабирања зато што уводи више нових идеја одједном:

- константну ниску смештену у меморију
- позив спољашње функције `puts`
- разлику између адресе ниске и садржаја на тој адреси
- бригу о стеку пре позива друге функције

Због тога је погоднији као други пример, након што су регистри, аргументи функције и повратна вредност већ познати из примера сабирања.

## Датотеке

- `hello.cpp` је референтна C++ верзија програма
- `hello.s` је асемблерска имплементација функције `main`

## Шта је ново у овом примеру

За разлику од сабирања, овде функција `main` не ради све сама, већ позива библиотечку функцију `puts`.

То значи да сада морамо да разумемо и следеће:

- текст `"Hello world!"` мора негде да буде смештен у меморији
- пошто је то константна ниска, природно је да буде у секцији `.rodata`
- адреса те ниске мора да се проследи функцији `puts`
- `lea` рачуна адресу ниске, а не чита њен садржај
- `call puts` уписује повратну адресу на стек и преноси управљање библиотечкој функцији
- пре позива друге функције морамо водити рачуна о стеку и његовом поравнању

У асемблерској верзији ниска је уписана у секцији `.rodata`:

```asm
.section .rodata
msg: .asciz "Hello world!"
```

Инструкција

```asm
lea rdi, [rip+msg]
```

израчунава адресу ниске и уписује је у регистар `rdi`, а то је управо место на које по конвенцији позива стиже први аргумент функције.

После тога позив

```asm
call puts
```

преноси управљање библиотечкој функцији која исписује ниску на стандардни излаз.

Важно је и да `puts` сам додаје нови ред на крају исписа, па у овом примеру сама ниска не садржи `\n`.

## Зашто овде постоји стек оквир

За разлику од `saberi`, функција `main` овде позива другу функцију. На самом уласку у `main` повратна адреса је већ на стеку, па `rsp` није поравнат исто као непосредно пре `call`.

Инструкције

```asm
push rbp
mov rbp, rsp
```

овде имају двоструку улогу:

- праве уредан стек оквир
- после `push rbp` враћају `rsp` на поравнање погодно за `call puts`

## Превођење

Ако желите да преведете C++ референтну верзију:

```sh
g++ hello.cpp
```

Ако желите да преведете асемблерску верзију:

```sh
g++ hello.s
```

## Покретање

```sh
./a.out
```

Очекивани излаз је:

```text
Hello world!
```

## На шта треба обратити пажњу

- `main` и даље враћа вредност преко регистра `eax`, па зато на крају постављамо `mov eax, 0`.
- Ово је први пример у ком наша функција позива другу функцију.
- `lea rdi, [rip+msg]` рачуна адресу ниске; да смо користили `mov`, добили бисмо нешто друго.
- `call puts` позива библиотечку функцију, а завршни `ret` враћа извршавање позиваоцу функције `main`.
- Због тога је природан увод у касније примере са сложенијим позивима и радом са стеком.
- За визуелно објашњење шта су `rbp`, `rsp` и стек оквир, погледајте одељак [Стек оквири](../00-uvodne-teme/01-organizacija-izvrsnog-koda.md).
