# Максимум два броја

Ово је први јасан пример у ком се један `if/else` из C++ кода директно преводи у поређење и условни скок.

На њему се види:

- како ради `cmp`
- како се користи условни скок `jle`
- како се у асемблеру организују `if` и `else` гране

## Шта програм ради

Програм учитава два `int` броја, позива функцију `max` и исписује већи од та два броја.

Референтна C++ верзија функције би могла изгледати овако:

```cpp
int max(int x, int y) {
    int return_value;
    if (x > y) {
        return_value = x;
    }
    else {
        return_value = y;
    }
    return return_value;
}
```

У овом коду постоје две могуће путање извршавања:

- ако је `x > y`, враћамо `x`
- у супротном, враћамо `y`

У C++-у то пишемо помоћу `if/else`, али у асемблеру немамо посебну конструкцију тог типа. Уместо тога, имамо низ инструкција које се иначе извршавају редом, једна за другом. Када желимо гранање, морамо да омогућимо да се део тог низа прескочи. То се ради инструкцијама скока.

Скокови могу бити:

- безусловни, као `jmp`
- условни, као `jle`

Безусловни скок увек пребацује извршавање на неку другу ознаку. У овом примеру, када се `if` грана изврши и вредност `x` упише као резултат, више не желимо да се настави у `else` грану, па је прескачемо инструкцијом `jmp`.

Условни скок пребацује извршавање само ако је одговарајући услов испуњен. У овом примеру не тестирамо директно услов `x > y`, већ радимо супротно: ако `x <= y`, прелазимо на `else` грану. Тако се `if` грана прескаче онда када њен услов није испуњен.

Образац зато изгледа овако:

```asm
cmp x, y
jle else_grana

; if grana
jmp kraj

else_grana:
; else grana

kraj:
```

Инструкција `cmp` је кључна јер она омогућава условно гранање. У овом примеру `cmp edi, esi` пореди `x` и `y`, а `jle` (`jump if less or equal`) после тога прелази на `else` грану ако је `x <= y`. Пошто радимо са типом `int`, овде користимо означено поређење.

Овај пример се врло јасно види и у [Compiler Explorer-у](https://godbolt.org/z/Mo9GWhe4v), где су делови кода обележени истим бојама.

## Како условни скок заиста ради

Битан детаљ претходног кода је то што се прво извршава инструкција поређења, а тек након тога инструкција условног скока. То значи да резултат поређења који добијамо инструкцијом `cmp` негде мора бити запамћен, а да условни скокови касније гледају те информације.

Уколико желимо да упоредимо бројеве `x` и `y`, природно је да посматрамо знак израза `x - y`. Ствари се мало компликују зато што процесор ради са фиксном ширином регистра, па приликом одузимања може доћи до прекорачења. Зато `cmp` не памти само да ли је резултат нула или негативан, већ поставља више битова у регистру `FLAGS`.

Информације које памти су:
- `SF` (`sign flag`) - поставља се ако је резултат негативан
- `ZF` (`zero flag`) - поставља се ако је резултат нула, односно ако су операнди једнаки
- `OF` (`overflow flag`) - поставља се ако је код означене аритметике дошло до прекорачења
- `CF` (`carry flag`) - користи се код неозначених поређења; сигнализира пренос или позајмицу

На основу вредности ових битова инструкције условног скока доносе одлуке. Начин доношења одлука можемо видети у следећој табели:

| Инструкција | Значење | Када скаче |
| --- | --- | --- |
| `je` / `jz` | једнако | `ZF = 1` |
| `jne` / `jnz` | није једнако | `ZF = 0` |
| `jl` | мање, за означене бројеве | `SF != OF` |
| `jle` | мање или једнако, за означене бројеве | `ZF = 1` или `SF != OF` |
| `jg` | веће, за означене бројеве | `ZF = 0` и `SF = OF` |
| `jge` | веће или једнако, за означене бројеве | `SF = OF` |
| `jb` | мање, за неозначене бројеве | `CF = 1` |
| `jbe` | мање или једнако, за неозначене бројеве | `CF = 1` или `ZF = 1` |
| `ja` | веће, за неозначене бројеве | `CF = 0` и `ZF = 0` |
| `jae` | веће или једнако, за неозначене бројеве | `CF = 0` |

У овом примеру најважнији је ред са `jle`: после `cmp edi, esi` тај скок се извршава онда када је `edi <= esi`, односно када је `x <= y`.

## Датотеке

- `main.cpp` чита два броја и позива `max`
- `max.s` садржи асемблерску имплементацију функције

## Превођење

```sh
g++ main.cpp max.s
```

## Покретање

```sh
./a.out
```

Пример интеракције:

```text
4 9
max(4, 9) = 9
```

## На шта треба обратити пажњу

- ово је један од најважнијих образаца за читање асемблера: `cmp` плус условни скок
- пример користи означено поређење, јер се ради са типом `int`
- исти образац ће се касније појављивати и у сложенијим функцијама са више грана
