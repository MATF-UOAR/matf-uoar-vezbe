# Дељивост са 4

Овај пример користи дељење да би донео логичку одлуку. Зато је добар непосредан наставак примера `Дељење`.

На њему се јасно види:

- како се остатак при дељењу користи као услов
- како функција из асемблера може директно да позове `puts`
- како се `cmp` и условни скок комбинују после `idiv`

## Шта програм ради

Програм учитава један `int` број и позива функцију `deljiv_4`, а сама функција из асемблера исписује одговарајућу поруку.

Референтна C++ верзија функције изгледала би овако:

```cpp
void deljiv_4(int x) {
    if (x % 4 == 0) {
        puts("Deljiv je");
    }
    else {
        puts("Nije deljiv");
    }
    return;
}
```

## Датотеке

- `main.cpp` чита број и само позива `deljiv_4`
- `deljiv.s` садржи асемблерску имплементацију функције `deljiv_4`

## Шта треба посматрати у асемблеру

Функција најпре припрема дељење:

```asm
mov esi, 4
mov eax, edi
cdq
idiv esi
```

После тога остатак се налази у `edx`. Зато је довољно:

```asm
cmp edx, 0
jne else_grana
```

Ако је остатак нула, функција учитава адресу поруке у `rdi` и позива `puts`:

```asm
lea rdi, [rip+poruka_deljiv]
call puts
```

У супротном се позива `puts` са другом поруком.

## Превођење

```sh
g++ main.cpp deljiv.s
```

## Покретање

```sh
./a.out
```

Пример интеракције:

```text
12
Deljiv je
```

## На шта треба обратити пажњу

- за тип `int` користимо означено дељење, па се зато појављује `cdq`
- аргумент за `puts` иде у `rdi`, па зато адресу стринга учитавамо помоћу `lea`
- поруке су смештене у `.rodata`, као и у `hello world` примеру
- ово није најбржи начин да се провери дељивост са `4`, али је педагошки користан јер утврђује рад са остатком
- после овог примера вреди самостално пробати и битовску варијанту засновану на провери два најнижа бита
