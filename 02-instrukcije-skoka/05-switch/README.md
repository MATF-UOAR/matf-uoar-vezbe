# `switch` гранање

Овај пример показује како се један `switch` из C++ кода може ручно превести у више поређења и скокова у асемблеру.

На њему се јасно види:

- како више `case` грана могу да воде у исти блок кода
- како се `default` грана своди на „ако ништа није погођено“
- како се `switch` може реализовати и без jump table приступа

## Шта програм ради

Програм учитава број месеца и позива функцију `broj_dana_u_mesecu`.

Референтна C++ верзија функције изгледала би овако:

```cpp
int broj_dana_u_mesecu(int mesec) {
    switch (mesec) {
    case 1:
    case 3:
    case 5:
    case 7:
    case 8:
    case 10:
    case 12:
        return 31;
    case 4:
    case 6:
    case 9:
    case 11:
        return 30;
    case 2:
        return 28;
    default:
        return -1;
    }
}
```

Ако је улаз неисправан, функција враћа `-1`.

## Датотеке

- `main.cpp` чита број месеца и исписује резултат
- `switch.s` садржи асемблерску имплементацију функције `broj_dana_u_mesecu`

## Шта треба посматрати у асемблеру

Сваки `case` постаје поређење и условни скок:

```asm
cmp edi, 1
je ima_31
cmp edi, 3
je ima_31
```

Више `case` вредности може да скаче на исту ознаку. Тако се добија исти ефекат као у C++ коду кад више `case` грана дели један `return`.

Ако ниједно поређење не успе, извршавање пада на:

```asm
mov eax, -1
```

То је овде `default` грана.


## Превођење

```sh
g++ main.cpp switch.s
```

## Покретање

```sh
./a.out
```

Примери интеракције:

```text
2
Broj dana: 28
```

```text
13
Neispravan mesec
```

## На шта треба обратити пажњу

- ово је педагошки најједноставнији облик превођења `switch` наредбе
- компајлер понекад за густо распоређене `case` вредности прави табелу скокова што може убрзати код, али овде то намерно не радимо
- образац са више `cmp` и `je` скокова је довољан за разумевање логике гранања
