# Сума

На овом примеру видимо како се имплементира `for` петља и како се користи инструкција множења `mul`.

## Шта програм ради

Програм учитава `unsigned` број `n` и исписује збир првих `n` природних бројева.

Референтна C++ функција може да изгледа овако:

```cpp
unsigned suma(unsigned n) {
    unsigned trenutna_suma = 0;
    for (unsigned i = 1; i <= n; i++) {
        trenutna_suma += i;
    }
    return trenutna_suma;
}
```

Структура `for` петље је скоро иста као структура `while` петље. Разлика је у томе што пре почетка морамо да иницијализујемо индекс (`i = 1`), а на крају тела петље да га увећамо (`i++`).

## Шта треба посматрати у асемблеру

Прво бирамо регистре за текућу суму и индекс:

```asm
    xor eax, eax   # eax = trenutna_suma
    mov ecx, 1     # ecx = i
```

Затим на почетку сваке итерације проверавамо да ли је време за излазак из петље:

```asm
for_petlja:
    cmp edi, ecx
    jb  for_petlja_kraj
```

Овде је услов записан као „ако је `n < i`, изађи из петље“, што је еквивалентно C++ услову `i <= n`.

У телу петље додамо текући индекс на суму, увећамо индекс и скочимо назад:

```asm
    add eax, ecx
    add ecx, 1
    jmp for_petlja
```

## Имплементација без петље

Сума првих `n` природних бројева једнака је `n * (n + 1) / 2`, па исти резултат можемо да покушамо да добијемо и без петље.

Ту нам је потребна инструкција `mul`. Слично као `div`, и она има један експлицитан операнд. Тај операнд множи са вредношћу из `eax`, а пун 64-битни производ смешта у `edx:eax`:

```asm
    mov eax, edi
    add edi, 1
    mul edi
```

Овде је важан детаљ да после `mul` у `edx:eax` већ имамо цео производ. Зато овај пут не смемо да радимо `xor edx, edx`: у примерима са дељењем смо нулирали `edx` зато што смо полазили од 32-битног дељеника у `eax`, а овде би та иста инструкција обрисала горњу половину већ израчунатог производа.

Пошто су `n` и `n + 1` узастопни бројеви, један од њих је увек паран. Зато је производ `n * (n + 1)` увек дељив са `2`, па дељење са `2` можемо да посматрамо као померање целог 64-битног резултата `edx:eax` за један бит удесно:

```asm
    shrd eax, edx, 1
```

Пошто функција ионако враћа само `unsigned`, на крају нам је битан само регистар `eax`. Зато је овде довољно да `shrd eax, edx, 1` израчуна доњих 32 бита вредности `n * (n + 1) / 2`.

Ако бисмо после `mul` ипак урадили `xor edx, edx`, изгубили бисмо горња 32 бита производа. Конкретан пример је `n = 65536`:

- исправан резултат је `2147516416`
- ако се `edx` постави на нула после `mul`, добија се `32768`

Могли бисмо да помислимо и да је довољно да после `mul` једноставно урадимо `div 2`, без нулирања `edx`. То би заиста делило цео 64-битни производ, али `div` тражи да количник стане у `eax`. Када математички резултат пређе `2^32 - 1`, процесор пријављује `divide error`.

То овде није пожељно, јер функција враћа `unsigned` и природно је да се понаша као и верзија са петљом, односно да резултат једноставно overflow-ује у 32 бита. На пример, за `n = 100000` верзија са петљом враћа `705082704`; директан `div` над целим `edx:eax` би ту прекинуо програм уместо да врати доњих 32 бита резултата.

Напомена: као и код дељења, постоји и варијанта за означене бројеве. `imul` служи за означено множење, док `mul` ради са неозначеним вредностима.

## Датотеке

- `main.cpp` чита један број и позива `suma`
- `suma.s` садржи асемблерску имплементацију функције са петљом
- `suma_proizvod.s` садржи асемблерску имплементацију функције без петље

## Превођење

За верзију са петљом:

```sh
g++ main.cpp suma.s
```

За верзију без петље:

```sh
g++ main.cpp suma_proizvod.s
```

## Покретање

```sh
./a.out
```

Пример интеракције:

```text
5

```

## На шта треба обратити пажњу

- `for` петља се овде своди на иницијализацију, проверу услова, тело и инкремент
- `mul` уписује пун производ у `edx:eax`, па после множења не смемо механички да нулирамо `edx`
- код дељења смо `edx` нулирали да бисмо направили 64-битни дељеник од 32-битне вредности; овде је `edx` већ део исправног резултата множења
- `xor edx, edx` pa `div 2` даје погрешан резултат, а директан `div 2` над целим `edx:eax` може да пријави `divide error` кад количник не стане у `eax`
- зато у верзији без петље узимамо доњих 32 бита количника помоћу `shrd`
