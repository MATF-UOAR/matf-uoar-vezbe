# Ојлер

На овом примеру видимо како једна асемблерска функција позива другу и о чему морамо да водимо рачуна када то радимо.

## Шта програм ради

Програм учитава `unsigned` број и исписује вредност Ојлерове функције. За дати `n`, она враћа број оних вредности `k` за које важи `1 <= k < n` и `nzd(k, n) = 1`.

Референтна C++ функција може да изгледа овако:

```cpp
unsigned ojler(unsigned n) {
    unsigned br_up = 0;
    for (unsigned i = 1; i < n; i++) {
        if (nzd(i, n) == 1) {
            br_up++;
        }
    }
    return br_up;
}
```

## Шта треба посматрати у асемблеру

Бројач вредности које су узајамно просте са `n` чувамо у `r9d`, а индекс петље у `ecx`:

```asm
    xor r9d, r9d    # r9d = br_up
    mov ecx, 1      # ecx = i
```

На почетку петље услов излаза проверавамо овако:

```asm
for_petlja:
    cmp ecx, edi
    jae for_petlja_kraj
```

Пошто радимо са `unsigned` вредностима, за услов `i < n` користимо unsigned поређење, односно скок `jae`.

У телу петље позивамо `nzd`, па пре `call` смештамо аргументе у `edi` и `esi`:

```asm
    mov esi, edi    # drugi argument = n
    mov edi, ecx    # prvi argument = i
    call nzd        # eax = nzd(i, n)
```

После тога проверавамо да ли је повратна вредност једнака `1`. Ако јесте, увећавамо бројач:

```asm
    cmp eax, 1
    jne nisu_uzajamno_prosti
    inc r9d

nisu_uzajamno_prosti:
    inc ecx
    jmp for_petlja

for_petlja_kraj:
    mov eax, r9d
```

## Шта морамо да сачувамо пре `call`

Када једна функција позива другу, важно је да разликујемо регистре које чува функција која позива другу (`caller-saved`) и регистре које чува позвана функција (`callee-saved`).

Регистри `rax`, `rcx`, `rdx`, `rsi`, `rdi`, `r8`, `r9`, `r10` и `r11` спадају у регистре које чува функција која позива другу (`caller-saved`). То значи да позвана функција сме да их промени. Ако позивајућој функцији неки од тих регистара и даље требају после `call`, она мора сама да их сачува.

У овом примеру су нам пре позива `nzd` битни `r9`, `rcx` и `rdi`, па их зато чувамо овако:

```asm
    push r9
    push rcx
    push rdi
    push rdi

    mov esi, edi
    mov edi, ecx
    call nzd

    pop rdi
    pop rdi
    pop rcx
    pop r9
```

Додатни `push`/`pop` пар није ту због чувања још једне вредности, већ због поравнања стека. Непосредно пре `call` инструкције `rsp` мора да буде дељив са `16`. Ако бисмо стек ручно поравнавали помоћу `sub rsp, 8` и `add rsp, 8`, овај додатни пар нам не би требао, али некоме је лакше да памти правило као „пре сваког `call` имамо паран број `push` инструкција“.

## Датотеке

- `main.cpp` чита један број и позива `ojler`
- `nzd.s` садржи асемблерску имплементацију функције `nzd`
- `ojler.s` садржи асемблерску имплементацију функције `ojler`

## Превођење

```sh
g++ main.cpp ojler.s nzd.s
```

## Покретање

```sh
./a.out
```

Пример интеракције:

```text
87
56
```

## На шта треба обратити пажњу

- ово је први пример у ком једна асемблерска функција позива другу асемблерску функцију
- регистре које чува функција која позива другу (`caller-saved`) морамо да сачувамо ако су нам и даље потребни после `call`
- код `for` петље са `unsigned` вредностима треба пазити да користимо одговарајући условни скок, овде `jae`
