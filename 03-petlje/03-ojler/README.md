# Ojler
На овом примеру ћемо видети како да позивамо већ имплементиран асемблерски програма из другог, као и ствари о којима
би требало да водимо рачуна кад то радимо.
## Шта програм ради
Програм учитава `unsigned` број и исписује вредност Ојлерове функције која за аргумент узима
дати број. Ојлерова функција за дати број `n` исписује број бројева `k` који су мањи од `n` и узајамно прости
с њим. Референта С++ функција може да изгледа овако:
```
unsigned ojler(unsigned n) {
    unsigned br_up=0;
    for(int i=1;i<n;i++)
`    {
        if(nzd(i,n)==1)
        br_up++;
    }
    return br_up;
}
```
Са знањем стекнутим из претходна два задатка можемо једноставно да имплементирамо ову функцију.
Изаберемо регистар да нам представља бројач (`br_up`), на пример `r9d`. Такође изаберемо индекс
унутар `for` петље (`i`), у овом случају `ecx`.
```
    xor r9d, r9d    # r9d = br_up

    mov ecx, 1      # ecx = i
```
Унутар саме петље на почетку напишемо проверу услова ( и одговарајући излаз из петље)
```
for_petlja:
    cmp ecx, edi
    je for_petlja_kraj
```
У телу петље позивамо функцију `nzd` коју смо написали у глави 01 , али наравно пре тога убацимо
одговарајуће вредности у регистре `edi` и `esi`
```
    mov esi, edi    # drugi arg  = n
    mov edi, ecx    # prvi arg = i
    call nzd        # eax = nzd(i, n)
```
Након тога следи `if/else` грана у којој проверавамо да ли је повратна вредност ( сместена у `eax`) једнака
јединици, тјст. да ли су бројеви `i` и `n` узајамно прости. Ако нису, скачемо у `else` грану, инкрементирамо 
индекс и скачемо на почетак петље. Ако јесу, пре тога само инкрементирамо бројач.
```
    cmp eax, 1
        jne nisu_uzajamno_prosti
        inc r9d

    nisu_uzajamno_prosti:


        inc ecx
        jmp for_petlja
    for_petlja_kraj:

        mov eax, r9d
```


## Шта недостаје у тренутној имплементацији?
Ако се подсетимо прве недеље, можемо да се сетимо да се регистри опште намене деле по томе да ли припадају позивајућој
или позваној функцији. Ако припадају позивајућој функцији, програм мора да обезбеди да ће њихова почетна вредност
бити очувана на крају програма (ово се најчешће реализује уз помоћ `push` и `pop` инструкција). Регистри који 
припадају позваној функцији немају ово ограничење, и након функције `call` могу бити измењени. Регистри `r11d`,`ecx`
и `edi` баш припадају позваној функцији, а у њима чувамо битне информације. Самим тиме требало би да их сачувамо на 
стеку пре него што позовемо функцију `nzd`.
```
    push r9
    push rcx
    push rdi
    push rdi

    mov esi, edi    # drugi arg  = n
    mov edi, ecx    # prvi arg = i
    call nzd        # eax = nzd(i, n)


    pop rdi 
    pop rdi
    pop rcx
    pop r9
```
Напомена: Не заборавимо да је неопходно "поравнати" стек тако да је адреса y `rsp` дељивa са 16 пре позива било које 
функције, зато радимо додатни `push` и `pop`(те функције могу да се замене са `sub rsp, 8` и `add rsp, 8` , али 
додатни `push` и `pop` раде посао). 

## Датотеке
- `main.cpp` чита један број и позива ојлер
- `nzd.s` садржи асемблерску имплементацију функције nzd
- `ojler.s` садржи асемблерску имплементацију функције ojler
## Превођење
  ```
g++ main.cpp ojler.s nzd.s 
  ```
